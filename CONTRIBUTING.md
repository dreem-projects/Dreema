# Contributing to Dreema

Thank you for your interest in contributing to Dreema.

Dreema is built around one core idea: **structure is a feature**. Contributions should preserve and strengthen that principle.

---

## The Dreema Philosophy

When contributing, prioritize:

- **Determinism** — Follow the established MVC architecture. Avoid ambiguous or inconsistent patterns.
- **Machine Readability** — All responses, errors, and interfaces must follow the Dreema standard contract.
- **Predictability** — Avoid hidden behavior, magic abstractions, or overly flexible designs that reduce clarity.

---

## How to Contribute

### 1. Reporting Issues

If you discover a bug or unexpected behavior:

- Check the existing [Issues](https://github.com/dreem-projects/Dreema/issues) to avoid duplicates.
- Provide a clear **error trace** using the Dreema standard format.
- Include a minimal, reproducible example.

Clear reports help both humans and AI systems diagnose issues faster.

---

### 2. Pull Requests

We welcome improvements across the framework.

To contribute:

- Fork the repository and create a feature branch.
- Follow the existing **project structure and MVC conventions**.
- Ensure all tests pass before submitting.
- Update relevant documentation when introducing changes.

Consistency is more important than creativity in core architecture.

---

### 3. Improving the Developer & AI Experience

Dreema is designed to improve both **developer productivity** and **AI-assisted development**.

We welcome contributions that make the framework better across all layers, including:

- `dreema-ai` workflows and agent integration
- error handling and parsing systems
- security and authentication improvements
- database layer and ORM enhancements
- routing, middleware, and architecture design
- overall developer experience and tooling

If your idea improves clarity, reliability, or productivity in Dreema, it is welcome.

We strongly encourage contributions that make the framework more predictable, secure, and enjoyable to build with.

---

## Code Style Guidelines

- **Explicit over implicit** — Code should be readable without context.
- **Type hints required** — We prioritize clarity for both developers and AI systems.
- **Consistent responses** — All endpoints must follow the Dreema Universal Response Contract.
- **No hidden behavior** — Avoid side effects that are not clearly defined.

---

## Contribution Agreement

By contributing to Dreema, you agree to maintain the structural integrity of the framework and follow its design principles.

This ensures Dreema remains predictable, scalable, and AI-friendly for everyone.
